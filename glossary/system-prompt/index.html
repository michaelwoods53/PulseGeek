System Prompt

A high-priority instruction that sets role, tone, and boundaries for the model, shaping behavior across an entire conversation or workflow.

In detail

A System Prompt is a top-level instruction that establishes the model’s role, goals, tone, constraints, and safety boundaries. Because it sits above user messages, it can shape responses across a session, improving consistency and reducing drift. Teams use system prompts to define style guides, compliance requirements, or task policies. Good system prompts are concise, testable, and paired with examples or rubrics where needed. They should be versioned, documented, and validated alongside other parts of the prompt stack to ensure reliable behavior at scale.

Synonyms

system message, meta-instruction

Mempool

The mempool holds unconfirmed transactions waiting to be added to a block. It affects ordering, fees, and attack surfaces like front-running and MEV.

In detail

The mempool is the set of unconfirmed transactions broadcast to the network but not yet included in a block. Nodes maintain mempools and share transactions with peers. Miners or validators select transactions from the mempool, often prioritizing higher fees. Ordering in the mempool can create risks such as front-running and sandwich attacks, especially for contracts that reveal pending actions. Developers can reduce exposure by using commit–reveal schemes, off-chain ordering, or private transaction relays. Understanding mempool behavior helps design more secure and predictable smart contract interactions.

Synonyms

transaction pool, pending transactions

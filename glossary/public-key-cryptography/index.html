Public Key Cryptography

Public key cryptography uses a key pair: a public key for verification or encryption and a private key for signing or decryption. It enables secure identity and communication in open networks.

In detail

Public key cryptography is built on asymmetric key pairs, where the public key can be shared widely and the private key must be kept secret. In blockchain systems, the public key or its hash often acts as an address, while the private key authorizes actions by producing valid signatures. Security comes from hard math problems, such as discrete logarithms on elliptic curves. This model avoids the need for shared secrets and supports secure interactions between strangers. It also enables multi-signature policies, threshold schemes, and advanced protocols like zero-knowledge proofs that reference public keys without revealing private information, improving both security and privacy in decentralized environments.

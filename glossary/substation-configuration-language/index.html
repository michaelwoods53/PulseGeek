Substation Configuration Language

An XML-based language in IEC 61850 for describing substation systems, IED capabilities, and communications. SCL files enable consistent engineering, interoperability, and testing across multi-vendor protection and control deployments.

In detail

Substation Configuration Language is the IEC 61850 engineering framework used to model and exchange substation designs in a structured, vendor-neutral way. SCL uses XML files to describe substation topology, primary equipment, logical nodes, IED capabilities, datasets, GOOSE and Sampled Values subscriptions, and communication settings. Common file types include ICD, SCD, CID, and IID, each serving a role in moving from device capability to a complete system configuration. This consistency allows utilities and integrators to design, simulate, test, and commission multi-vendor systems with less manual rework and fewer integration errors. SCL supports lifecycle activities, from factory acceptance testing to site commissioning and maintenance, by preserving a single source of truth for logical and physical configurations. It also eases coordination with higher level enterprise systems by aligning semantic data modeling, and it underpins fast messaging schemes like GOOSE for protection and interlocking functions.

Synonyms

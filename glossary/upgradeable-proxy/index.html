Upgradeable Proxy

An upgradeable proxy lets a contract’s logic be replaced while keeping the same address and state, enabling fixes and improvements after deployment.

In detail

An upgradeable proxy pattern separates a contract’s storage and address from its logic implementation. Users interact with the proxy, which delegates calls to an implementation contract. When an update is needed, the proxy can be pointed to new logic while preserving state and the public address. This approach supports bug fixes, feature additions, and evolving standards, but it adds governance risks and complexity. Secure upgrades require access control, timelocks, and well-tested migration paths. Common models include transparent proxies and UUPS, each with trade-offs in flexibility and safety.

Synonyms

upgradeable contracts, proxy pattern

Abbreviations

UUPS

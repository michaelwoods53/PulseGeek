Reentrancy

Reentrancy is a vulnerability where a contract’s external call lets another contract call back before state updates finish, enabling repeated withdrawals or logic misuse.

In detail

Reentrancy occurs when a contract makes an external call and the callee calls back into the caller before it has finalized state changes. This can allow repeated execution of sensitive functions, such as withdrawing funds multiple times. Common defenses include checks-effects-interactions patterns, reentrancy guards, pull-payment models, and minimizing external calls. Testing with adversarial cases and using static analysis tools can help catch reentrancy risks. Understanding call flow, storage updates, and error handling is essential to prevent this classic vulnerability.

Synonyms

reentrant call, reentrancy bug
